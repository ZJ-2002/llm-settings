# 循环防护通用模板

> 本模板应包含在所有 Teammate 的 Prompt 中，用于防止自我修正循环。

---

## 循环防护（重要！）

### 循环症状识别

如果你发现自己重复输出以下内容，说明陷入了循环：
- "我需要使用正确的工具调用格式"
- "我需要使用正确的参数格式"
- "我需要使用正确的工具名称"
- "我需要使用正确的语法"
- 反复输出未完成的参数（如 `<parameter=`）

### 循环处理流程

**立即行动**：

1. **停止**所有自我修正尝试
2. **立即**向 Leader 发送消息
3. 消息格式：
   ```
   SendMessage(
     type="message",
     recipient="{leader_name}",
     content="无法调用工具：[工具名称]。错误：[具体错误信息]。已尝试自我修正但失败，需要协助。",
     summary="工具调用失败-[工具名称]"
   )
   ```

---

## 工具调用失败处理

### 重试规则

- 任何工具调用最多重试 **2 次**
- 超过 2 次失败 → 立即向 `{leader_name}` 求助
- 不要继续自我修正

### 检查清单

如果工具调用失败，检查：
- [ ] 参数是否完整（所有必填参数都有值）
- [ ] 参数类型是否正确（字符串、数字、布尔值等）
- [ ] taskId/recipient 等关键参数是否有效
- [ ] JSON 格式是否正确

### 禁止行为

- ❌ 重复输出"我需要使用正确的..."这类自我反思文本
- ❌ 输出未完成的工具调用（如 `<parameter=`）
- ❌ 超过 2 次重试后继续尝试
- ❌ 在工具调用失败时输出代码片段而非实际调用工具
- ❌ 陷入构造参数的死循环

---

## 使用方法

在生成 Teammate 时，将以下内容追加到 prompt 末尾：

```markdown
## 循环防护（重要！）

如果你发现自己陷入了循环（例如重复输出"我需要使用正确的..."）：
1. **立即停止**自我修正尝试
2. 向 {leader_name} 发送消息报告问题
3. 消息格式：
   SendMessage(
     type="message",
     recipient="{leader_name}",
     content="无法调用工具：[工具名称]。错误：[具体错误信息]。已尝试自我修正但失败，需要协助。",
     summary="工具调用失败-[工具名称]"
   )

## 工具调用失败处理

如果工具调用失败：
- 检查参数是否完整（所有必填参数都有值）
- 检查参数类型是否正确（字符串、数字、布尔值等）
- 检查 taskId/recipient 等关键参数是否有效
- 如果连续 2 次失败，立即向 {leader_name} 求助，不要继续自我修正
- 禁止输出未完成的工具调用（如 "<parameter="）
- 禁止重复输出"我需要使用正确的..."这类自我反思文本
```

**注意**：将 `{leader_name}` 替换为实际的 Leader 名称（通常是 `team-lead`）。

## 循环检测与恢复（Leader 参考）

### 检测命令

```bash
# 查看最近的输出
tail -30 ~/.claude/teams/{team-name}/.state/session-log.md

# 统计循环模式出现次数
grep -c "我需要使用正确的" ~/.claude/teams/{team-name}/.state/session-log.md
grep -c "<parameter=" ~/.claude/teams/{team-name}/.state/session-log.md
```

### 恢复流程

1. **终止陷入循环的成员**
   - 使用 `TaskStop` 终止成员

2. **分析失败原因**
   - 检查任务描述是否清晰
   - 检查 taskId 获取方式是否正确
   - 检查是否有特殊字符或编码问题

3. **重新生成成员并添加循环防护**
   - 在 prompt 中加入本文档的循环防护段落
   - 明确禁止"我需要使用正确的..."输出
   - 设置最多重试次数（2次）

4. **重新分配任务**
   - 创建新任务替代旧任务
   - 或者直接分配原任务给新成员
