# GRADE 证据质量评估框架

## 概述

GRADE（Grading of Recommendations Assessment, Development and Evaluation）是国际标准化的证据质量评估和推荐强度分级系统。本框架用于将传统"证据等级"提升至"证据确定性"和"推荐强度"的完整评价体系。

---

## 第一部分：证据确定性评估

### 初始证据等级

| 研究设计 | 初始等级 | 说明 |
|----------|----------|------|
| 随机对照试验（RCT） | 高 | 起始为高确定性 |
| 观察性研究 | 低 | 起始为低确定性 |

### 降级因素（5大因素）

评估每项证据时，必须检查以下五个降级因素：

#### 1. 偏倚风险（Risk of Bias）

**评估标准**：
- **严重问题**：降级1级
  - 随机化方法不当或分配隐藏缺失
  - 盲法缺失导致测量偏倚
  - 失访率>20%且无恰当处理
  - 选择性报告结果
  
- **非常严重问题**：降级2级
  - 上述多个问题同时存在
  - 关键设计缺陷导致结果不可信

**记录模板**：
```markdown
偏倚风险评估：
- 降级级别：0级 / -1级 / -2级
- 主要问题：[具体问题描述]
- 影响方向：[可能夸大/缩小效应]
- 证据来源：[RoB 2 或 NOS 评估结果]
```

#### 2. 不一致性（Inconsistency）

**评估标准**：
- **严重不一致**：降级1级
  - I²统计量 > 50%（异质性中等）
  - 效应方向不一致（有的研究显示获益，有的显示危害）
  - 效应量差异显著（如RR从0.5到2.0）
  
- **非常严重不一致**：降级2级
  - I²统计量 > 75%（异质性高）
  - 各研究结论完全相反
  - 无法解释的异质性

**记录模板**：
```markdown
不一致性评估：
- 降级级别：0级 / -1级 / -2级
- I²值：[具体数值]
- 效应方向一致性：[一致/部分一致/不一致]
- 可能解释：[人群差异、干预差异、结局差异等]
```

#### 3. 间接性（Indirectness）

**评估标准**：
- **严重间接性**：降级1级
  - **人群间接**：研究人群与目标人群差异大
    - 示例：研究在年轻人中进行，但需推断老年人
  - **干预间接**：实际干预与理想干预差异大
    - 示例：研究使用口服药，但需推断注射剂
  - **对照间接**：对照组与临床实际差异大
    - 示例：对照为安慰剂，但临床实际使用活性对照
  - **结局间接**：替代终点而非临床终点
    - 示例：使用生物标志物替代硬终点（死亡率）
    
- **非常严重间接性**：降级2级
  - 上述多个问题同时存在
  - 关键问题导致适用性极低

**记录模板**：
```markdown
间接性评估：
- 降级级别：0级 / -1级 / -2级
- PICO匹配度：
  - Population：[直接/部分间接/间接]
  - Intervention：[直接/部分间接/间接]
  - Comparison：[直接/部分间接/间接]
  - Outcome：[直接/替代终点/高度替代]
- 适用性判断：[高/中/低]
```

#### 4. 不精确性（Imprecision）

**评估标准**：
- **严重不精确**：降级1级
  - 样本量小（每组<100例）
  - 95%CI跨临床决策阈值
  - 事件数少（<300例）
  
- **非常严重不精确**：降级2级
  - 样本量极小（每组<50例）
  - 95%CI极宽，包含获益和危害
  - 最优信息规模（OIS）未达到

**最优信息规模（OIS）计算**：
```markdown
OIS计算要素：
- 对照组事件率：[具体数值]
- 预期相对效应（RR/RD）：[具体数值]
- α水平（通常0.05，双侧）
- β水平（通常0.20，即检验效能80%）
- 计算结果：需要 [X] 例患者
- 实际纳入：[Y] 例
- 结论：[达到/未达到OIS]
```

**记录模板**：
```markdown
不精确性评估：
- 降级级别：0级 / -1级 / -2级
- 总样本量：[X] 例
- 事件数：[Y] 例
- 95%CI：[具体数值]
- 决策阈值：[具体数值]
- CI跨阈值：[是/否]
- OIS评估：[达到/未达到]
```

#### 5. 发表偏倚（Publication Bias）

**评估标准**：
- **强烈怀疑**：降级1级
  - 漏斗图不对称
  - Egger检验 P < 0.10
  - 小样本研究效应明显
  - 阴性结果未发表
  
- **非常强烈怀疑**：降级2级
  - 上述证据确凿
  - 存在注册研究未发表的情况
  - 药企资助研究显示显著效应

**记录模板**：
```markdown
发表偏倚评估：
- 降级级别：0级 / -1级 / -2级
- 漏斗图分析：[对称/轻度不对称/明显不对称]
- 统计检验：[Egger test P值]
- 小样本研究效应：[存在/不存在]
- 未发表研究：[有注册但未发表/无]
- 其他证据：[如行业资助偏倚]
```

### 升级因素（观察性研究）

对于观察性研究，存在以下升级因素时可提升证据等级：

| 升级因素 | 升级级别 | 判断标准 |
|----------|----------|----------|
| 大效应量 | +1或+2级 | RR > 2 或 RR < 0.5（+1级）；RR > 5 或 RR < 0.2（+2级） |
| 剂量-反应关系 | +1级 | 存在明确的剂量-效应梯度 |
| 残余混杂可能降低效应 | +1级 | 所有已知混杂都会低估真实效应 |

### 最终证据确定性分级

| 等级 | 描述 | 代码 | 含义 |
|------|------|------|------|
| 高 | 非常有把握 | ⊕⊕⊕⊕ | 真实效应值接近估计值 |
| 中 | 中等把握 | ⊕⊕⊕◯ | 真实效应值可能接近估计值，但也可能差异较大 |
| 低 | 有限把握 | ⊕⊕◯◯ | 真实效应值可能与估计值差异较大 |
| 极低 | 几乎没有把握 | ⊕◯◯◯ | 真实效应值可能与估计值差异极大 |

## 第二部分：推荐强度形成

### Evidence-to-Decision (EtD) 框架

形成推荐时必须考虑以下因素：

#### 1. 问题优先级
- **评估**：该问题在健康/临床实践中的重要性
- **记录**：高/中/低优先级，说明理由

#### 2. 效果确定性
- **评估**：基于上述GRADE评估
- **记录**：⊕⊕⊕⊕ / ⊕⊕⊕◯ / ⊕⊕◯◯ / ⊕◯◯◯

#### 3. 效果平衡（获益 vs 危害）
- **评估**：
  - 明确获益 > 危害 → 强烈推荐
  - 获益可能 > 危害 → 条件推荐
  - 获益 ≈ 危害 → 无明确推荐
  - 危害 > 获益 → 强烈不推荐

#### 4. 资源利用和成本效益
- **评估**：干预的经济可及性
- **记录**：高成本效益 / 中等 / 低成本效益

#### 5. 价值观念与偏好
- **评估**：患者/临床医生的价值取向
- **记录**：相对一致 / 可能存在差异 / 差异显著

#### 6. 可接受性和可行性
- **评估**：干预在实际场景中的可操作性
- **记录**：高可接受性 / 中等 / 低可接受性

### 推荐强度分级

| 强度 | 表述 | 含义 |
|------|------|------|
| **强烈推荐** | "推荐..." | 绝大多数患者应选择此干预 |
| **条件推荐** | "建议..." | 多数患者应选择此干预，但需个体化决策 |

## 第三部分：GRADE 证据概要表（SoF表）

### 标准格式

| 结局指标 | 参与者人数（研究数） | 相对效应（95%CI） | 绝对效应（95%CI） | 证据确定性 | 重要性 |
|----------|---------------------|-------------------|-------------------|------------|--------|
| 主要结局1 | XXX（X项RCT） | RR X.XX (X.XX-X.XX) | XX/1000（XX-XX） | ⊕⊕⊕◯ 中等 | 关键 |
| 主要结局2 | XXX（X项RCT） | RR X.XX (X.XX-X.XX) | XX/1000（XX-XX） | ⊕⊕◯◯ 低 | 关键 |
| 次要结局1 | XXX（X项观察性） | - | - | ⊕⊕◯◯ 低 | 重要 |

### 脚注说明

a. 降级原因：偏倚风险（X项研究随机化不当）
b. 降级原因：不一致性（I²=75%，异质性高）
c. 降级原因：不精确性（样本量不足，CI宽）

## 第四部分：实施流程

### STEP 1: 证据分级

对每个关键结局进行：
1. 确定研究设计类型
2. 应用5大降级因素
3. 观察性研究考虑3大升级因素
4. 确定最终证据确定性等级

### STEP 2: 证据概要表制作

1. 选择关键结局（通常3-7个）
2. 提取效应量和95%CI
3. 计算绝对效应（如适用）
4. 标注证据等级和降级原因

### STEP 3: 形成推荐

使用EtD框架：
1. 列出所有决策因素
2. 小组讨论形成共识
3. 确定推荐强度和方向
4. 撰写推荐说明和注意事项

大纲设计时规划：
- 哪些章节需要GRADE证据表
- 关键推荐的呈现位置
- 证据确定性与论述强度的匹配

### 与 STEP 8 (review-checklist) 集成

新增检查项：
- [ ] 关键证据是否使用GRADE评估？
- [ ] 证据概要表是否完整？
- [ ] 推荐强度是否有EtD支持？
- [ ] 降级/升级因素是否说明？
