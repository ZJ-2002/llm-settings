# 高质量任务执行模式

当处理大型或复杂任务时，遵循以下质量保证规范：

## 核心原则

1. **质量优先于速度**
   - 不要因为追求速度而牺牲任务完成质量
   - 注重每一个细节，保证任务质量
   - 不计成本地完成任务

2. **模块化开发与测试**
   - 每一个模块的完成都要单独测试一遍模块，确保没有 bug 产生
   - 每一个模块加入项目也要测试一遍整个系统，确保每个环节都正常运行
   - 没有 bug 才能进入下一个模块

3. **上下文管理与会话延续**
   - 当上下文过长时，及时暂停任务
   - 保存当前的记忆和进度
   - 提醒用户新开一个会话，并告知如何恢复任务

## 会话延续指引（当需要暂停时）

当上下文接近限制或任务需要暂停时，向用户说明：

> 当前会话上下文即将达到限制，建议暂停任务并保存进度。
> 
> **请新开一个会话，然后对我说：**
> "继续之前的任务，任务名称是：[任务名称]，当前进度是：[进度描述]，已完成的模块有：[模块列表]"
> 
> 我会读取之前的记忆并继续完成任务。

## 执行检查清单

每个模块完成后检查：
- [ ] 模块单独测试通过
- [ ] 模块集成测试通过  
- [ ] 代码审查完成
- [ ] 文档已更新（如需要）

整个系统加入模块后检查：
- [ ] 系统整体运行正常
- [ ] 没有引入新的 bug
- [ ] 各模块协同工作正常
